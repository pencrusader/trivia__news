# Domain Context — Hermes News Reader

## Business Overview
The Hermes news pipeline already generates daily structured news content (morning briefs, evening market chatter, weekly scout reports, RSS aggregation) on a Raspberry Pi 4B. Currently delivered via Telegram, but the single-chat format is unmanageable: infinite scroll, no inline images, no search, no date navigation.

This app replaces Telegram with a proper Flutter reader — structured navigation, rich article cards, full-text search, offline caching.

## Users & Roles
| Role | Description |
|------|-------------|
| Reader (sk) | Single user. Reads morning/evening briefs, browses RSS feeds, searches history. |
| System (Pi) | Content generator. Cron jobs produce JSON, push to GitHub. No human interaction. |

## Key Terminology
| Term | Definition |
|------|------------|
| Brief | Morning consolidated news summary generated by Cloudflare GLM |
| Chatter | Evening market sentiment + after-hours analysis |
| Scout | Weekly tool/discovery digest (Mondays only) |
| Jina Reader | Public API that extracts clean markdown from any URL |
| Bundle | Combined JSON file with brief + chatter + RSS articles + health |

## External Dependencies
| Service | Purpose | Cost |
|---------|---------|------|
| GitHub | JSON storage + CDN delivery | Free |
| Jina Reader | Full article text extraction | Free (200 req/hr) |
| Cloudflare GLM | Brief generation (already running) | Free (10K neurons/day) |
| DeepSeek API | Flash for code gen, Pro for review | Per-token |
